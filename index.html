General Guidelines

  1. Keep the Widget and add-on simple. Do one task and do it well.

  2. Your repository will be named "collective.widgetname". This will allow us to import your widget by using the following import statement::

    from kivy.collective.widgetname import WidgetName

  3. Name your widget correctly. Don't put "Widget" in the widget name as this is obvious. (SliderWidget, ProgressBarWidget are wrong, keep it simple.)

  4. Create adequate documentation that explains the functionality of your widget.

  5. Follow the PEP8 guidelines for coding standards.

  6. Follow the kivy guidelines for contributing. To install the pep8 checker into your git directory do

    cp path-to-kivy/kivy/tools/pep8checker/pre-commit.githook path-to-your-source/.git/hooks/pre-commit chmod +x path-to-collective-source/.git/hooks/pre-commit

    Then assuming you have kivy in your path, our style guide check will run whenever you do a commit, and if there are violations in the parts that you changed, your commit will be aborted. Fix & retry.

  7. Use the Monitor module to ensure that your widget can maintain a FPS rate above 60, ensuring smooth interaction with the user.

  8. Follow the performance guidelines for Python, like:: ensuring minimim . lookups, avoiding .lookups in loops...

  9. As a way of self-organizing widgets, add your widget to the showcase[linkTODO] that best fits the subject area. If no showcase yet exists for a topic, open a ticket to make a request. We'd like to keep the number of showcases to a minimum, but know that it will take additional categories as the number of widgets grows.

  10. Use tags instead of a directory structure to categorize your widgets in the documentation.

Note: you must have a GitHub account to open a ticket. You can create one for free here. New repository.

Disabling notification spam

By default you are subscribed to all new collective github repository notifications. Unless you disable this yourself, your inbox will soon contain only Github spam.

Go to https://github.com/watching Select what to watch or uncheck Automatically watch. But attention: latter will disable watch for all organizations.

Transferring repository

How to move your repository to Kivy Collective shared ownership?

About transferring the ownership of repositories. See Github instructions regarding transfers

Personally contact any of the Kivy Collective administrators Make him/her the owner of the repository which needs to be transferred. If the repository is already part of Github organization then create a new team and make the collective admin person the sole member of this team. Then assign the ownership permission of the repository to the new team. Now the collective administrator can go to the repository settings and press Transfer Ownership button
